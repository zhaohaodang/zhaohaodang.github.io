30 秒就可以理解的 javascript 代码片段

这些代码片段常常被用在热门的第三方库中，理解它们有助于搞懂第三方源码。

高阶函数 ​

ary ​

创建一个只接受 n 个参数的函数，忽略其他参数。

使用展开操作符（...）将参数聚合到数组 args，使用切片方法 Array.prototype.slice(0,n) 截取 args 前 n 个参数。

js
 const ary = (fn, n) => (...args) => fn(...args.slice(0, n));
查看示例
js
const add = (a, b, c) => a + b + c;

// 使用 ary 限制参数个数为 2
const addTwoArgs = ary(add, 2);

console.log(addTwoArgs(1, 2, 3)); // 输出 3，而不是 6

call ​

给定一个键名 key 和一些参数，返回可以接受一个上下文的函数，该函数用于调用上下文中的 key 方法，并传入参数。

js
const call = (key, ...args) => context => context[key](...args);
查看示例
js
const obj = {
  greet: (name) => `Hello, ${name}!`,
};

const greetCaller = call("greet", "Alice"); // 生成一个调用 `greet` 方法的函数
console.log(greetCaller(obj)); // 输出：Hello, Alice!

collectInto ​

定义一个高阶函数，将多个参数收集到一个数组中，然后将该数组作为单个参数传递给目标函数。

js
const collectInto = fn => (...args) => fn(args);
查看示例
js
// 示例函数，计算数组的平均值
const average = (arr) => arr.reduce((sum, num) => sum + num, 0) / arr.length;

const collectAverage = collectInto(average);

console.log(collectAverage(10, 20, 30, 40)); // 输出：25

flip ​

定义一个高阶函数，翻转目标函数的参数顺序。将传入的第一个参数移到最后，其余参数保持原顺序。

通常用于需要调整参数顺序的场景，尤其是当函数的参数顺序与实际使用习惯不符时。

js
 const flip = fn => (first, ...rest) => fn(...rest, first);
查看示例
js
// 示例函数，判断元素是否在数组
const includes = (array, element) => array.includes(element);
//

over ​

创建一个高阶函数，接受一组函数作为参数，返回一个新函数，调用新函数的时候，将传入的参数依次应用到这组函数上，并返回这些函数调用结果的数组。

场景：当需要对同一组输入应用多种操作时，比如数据计算、格式转换等。

js
const over = (...fns) => (...args) => fns.map(fn => fn.apply(null, args));
查看示例
js
const applyFns = over(Math.max, Math.min);
applyFns(3, 2, 7); // [7, 2]

overArgs ​

定义一个高阶函数，接受一个目标函数，和一组转换函数，对传入的参数依次使用转换函数，转换后再调用目标函数。

js
const overArgs = (fn, transforms) => (...args) => fn(...args.map((val, i) => transforms[i](val)));
查看示例
js
const multiply = (a, b) => a * b;

const double = (n) => n * 2;
const square = (n) => n * n;

const newFn = overArgs(multiply, [double, square]);

console.log(newFn(2, 3)); // 转换后的参数是 2*2 和 3*3, 输出 4 * 9 = 36

这种模式非常适合在函数调用前对参数进行预处理。

pipeAsyncFunctions ​

定义一个高阶函数，接受一组异步函数，按顺序依次执行，将每个异步函数的输出作为下一个函数的输入。

js
const pipeAsyncFunctions = (...fns) => (arg) => fns.reduce((p, fn) => p.then(fn), Promise.resolve(arg));
查看示例
js
const asyncAdd = async (x) => x + 2;
const asyncMultiply = async (x) => x * 2;

const newFn = pipeAsyncFunctions(asyncAdd, asyncMultiply);

newFn(1).then(console.log);

应用场景

  1. 数据流处理：
    • 用于对输入数据进行多步异步转换。
    • 类似于管道操作，每一步依赖于前一步的结果。
  2. 中间件机制：
    • 在请求处理或任务执行中，按顺序执行多个异步中间件函数。
  3. 函数式编程：
    • 结合 pipe 和异步处理，实现更优雅的函数组合。

pipeFunctions ​

定义一个高阶函数，接受一组函数，返回一个新函数，这个新函数能够将输入，从左到右，依次传递给每个函数，上一个函数的输出，是下一个函数的输入，最终返回最后一个函数的结果。

js
const pipeFunctions = (...fns) => fns.reduce((f, g) => (...args) => g(f(...args)));
查看示例
js
const add1 = (x) => x + 1;
const double = (x) => x * 2;

const add1AndDouble = pipeFunctions(add1, double);

add1AndDouble(3); // 8

如果想从右向左执行每个函数，只需要交换累计结果 f 和当前函数 g 的执行顺序。

js
const pipeFunctionsRTL = (...fns) =>
  fns.reduce(
    (f, g) =>
      (...args) =>
        f(g(...args))
  );
查看示例
js
const add1 = (x) => x + 1;
const double = (x) => x * 2;

const doubleAndAdd1 = pipeFunctionsRTL(add1, double);

doubleAndAdd1(3); // 7

这种模式适合将简单的方法组合成复杂的操作。

promisify ​

在 Node.js 和 JavaScript 中，许多旧的 API 使用了回调函数（callback）来处理异步操作，比如 fs.readFile ：

js
fs.readFile("example.txt", (err, data) => {
  if (err) {
    console.error(err);
    return;
  }
  console.log(data.toString());
});

现在我们要写一个高阶函数，将一个传统的回调风格的异步函数，转换成 Promise 风格。

这么做的好处是可以避免“回调地狱”，还能使用async/await语法，简洁优雅。

js
const promisify =
  (fn) =>
  (...args) =>
    new Promise((resolve, reject) =>
      fn(...args, (err, result) => (err ? reject(err) : resolve(result)))
    );
查看示例
js
const fs = require("fs");
// 使用 promisify 转换
const readFile = promisify(fs.readFile);
(async () => {
  try {
    const data = await readFile("example.txt", "utf8");
    console.log(data);
  } catch (err) {
    console.error(err);
  }
})();

更标准的做法是使用 Node.js 原生提供的 util.promisify：

js
const { promisify } = require("util");
const readFile = promisify(fs.readFile);

rearg ​

定义一个高阶函数，接受一个目标函数fn, 和一个数组，数组定义了参数的新顺序，用于重新排列参数，然后传递给目标函数fn

js
const rearg = (fn, indexes) => (...args) => fn(indexes.map(i => args[i]));
查看示例
js
const fn = (a, b, c) => [a, b, c];

const reargFn = rearg(fn, [2, 0, 1]);

console.log(reargFn("x", "y", "z")); // 输出：['z', 'x', 'y']

数组 ​

all ​

定义一个函数all，接收一个数组 arr，和一个可选函数fn（默认为 Boolean），返回一个布尔值，表示数组中所有元素是否都满足fn 的判定条件。

js
const all = (arr, fn = Boolean) => arr.every(fn);
查看示例
js
all([1, 2, 3]); // true
all([1, null, 3]); // false

all([1, 2, 3], (x) => x > 2); // false

为什们不直接用数组的every方法，反而要包装一下，不是多此一举吗？

首先，all 这类单词更符合我们的直觉。

其次，all 方法提供了默认函数 Boolean，当需要频繁判断元素是否为真时，非常有用，省去很多冗余代码。

另外，可以在 all 方法内部新增一些通用的功能，比如新增调试日志。

js
const all = (arr, fn = Boolean) => {
  console.log(`Checking array: ${JSON.stringify(arr)}`);
  return arr.every(fn);
};

allEqual ​

判断数组中的元素是否都相等

js
const allEqual = (arr) => arr.every((v) => v === arr[0]);
示例
js
allEqual([1, 2, 3]); // false
allEqual([1, 1, 1]); // true

any ​

定义一个函数any，接收一个数组 arr，和一个可选函数fn（默认为 Boolean），返回一个布尔值，表示数组中某个元素是否满足函数fn 的判定条件。

js
const any = (arr, fn = Boolean) => arr.some(fn);
示例
js
any([1, 2], (x) => x > 1); // true
any([1, 2], (x) => x > 3); // false
